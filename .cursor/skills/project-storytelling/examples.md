# 示例：project-storytelling

## 示例 1：回答“为什么做这个项目”

**触发词**
- 为什么要做这个项目
- 你的项目动机是什么

**标准输出（短版）**
我做这个项目是为了解决非设计用户做图门槛高、流程割裂的问题。  
我把找图、编辑、AI 生图和 AI 抠图整合在一个在线编辑器里，提升从想法到成图的效率。  
这个项目也让我完整实践了前端编辑器架构和 AI 接口工程化接入。

## 示例 2：回答“这个项目有什么用”

**触发词**
- 项目价值是什么
- 这个项目落地意义在哪

**标准输出（要点）**
1. 用户价值：降低设计门槛，普通用户也能快速产出视觉内容。  
2. 业务价值：减少多平台切换，提高内容生产效率。  
3. 技术价值：形成可扩展的编辑器底座，可继续接模板、协作、云端存储。

## 示例 3：30 秒项目介绍

**触发词**
- 30秒介绍一下你的项目
- 简单讲一下你做的项目

**标准输出**
我做了一个 Canva 类在线编辑器，技术上基于 Next.js 和 Fabric.js。  
核心能力包括画布编辑、撤销重做、素材库接入，以及 AI 生图和 AI 抠图。  
后端用 Hono 做统一 API 层，前端通过 React Query 管理请求状态。  
我主要负责编辑器核心交互和 AI 能力接入，目标是让非设计用户也能高效做图。

## 示例 4：高频追问“你的难点是什么”

**触发词**
- 项目最难的点是什么
- 你怎么解决技术难题

**标准输出**
最难的是画布状态一致性和 AI 异步任务稳定性。  
前者我通过命令式 editor 层统一管理对象变更和状态读取，避免逻辑散落在组件。  
后者我做了入参校验、轮询上限、超时与错误映射，保证第三方波动时用户仍有清晰反馈。
