# 示例：ai-endpoint-resilience

## 示例 1：新增 AI 图像扩图接口

**触发词**
- 新增 `/api/ai/outpaint`
- 接入新的图像编辑模型

**标准输出**
1. 在 Hono 路由中新增 endpoint，并使用 `zod` 校验入参。
2. 校验 `API_KEY`，缺失时返回 `{ error }` 和 500。
3. 若为异步任务，补齐轮询间隔、最大次数、超时错误。
4. 前端用 React Query mutation 封装调用，处理 `response.ok` 分支。
5. UI 显示 `isPending/isError`，成功后写入画布。

## 示例 2：抠图接口偶发超时

**触发词**
- remove-bg 经常超时
- 轮询太久用户无反馈

**标准输出**
1. 检查轮询常量：间隔与最大次数是否合理。
2. 在超时分支返回可识别错误（如 `Remove background timed out`）。
3. 前端将超时错误映射为可理解提示，建议用户重试。
4. 加日志：任务创建失败、状态失败、结果缺失分别打印。
5. 回归：模拟慢响应，确认前端不会一直 loading。

## 示例 3：第三方返回结构变更

**触发词**
- AI 服务返回字段变了
- 前端拿不到 `data`

**标准输出**
1. 对第三方响应做显式结构解析，不直接链式读取到底。
2. 缺失关键字段时返回统一错误 `{ error: "Failed to ..."}`
3. 前端错误处理优先展示后端 message，兜底通用提示。
4. 增加最小回归测试场景：成功、失败、结构缺失三类。
